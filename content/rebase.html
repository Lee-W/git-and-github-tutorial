假設我們多了兩個commit
其中有一個commit只是修復打錯字
我們希望能把它整併進上一個commit

rebase


git rebase -i "SHA1"
找出SHA1這個commit以後（不包含這個commit）的commit進行rebase

接著就會進入rebase的互動介面
下面有說明可以進行哪些動作

rebase

假設我們只有要讓typo的commit整併到上一個commit 只要將pick改成f，並存檔


再去看log就會發現typo這個commit不見了

rebase

rebase甚至可以交換commit的順序
只要在剛剛的interactive mode調換commit的順序
就會更改在history中的順序
不過要注意的是，如果這些commit有修改到同樣的檔案，可能會造成問題

rebase

如果遇到conflict，可以用 

git rebase --abort
放棄這次的rebase

Pull with rebase


git pull --rebase
它可避免pull遠端branch時，造成無謂的merge節點
