Quick Start

Create a local repo

  • 什麼是repo?
    • repository，git都是這麼稱呼它的版本庫
    • 可以把它想成就是你要加入版本控制的專案

  • 初始一個git專案(讓這個資料夾被git記錄)

git init "path"
如果不輸入path，就會初始在目前所在的資料夾

Create a local repo

  • .git
    • 記錄git所有的操作
    • 裡面都是一般的file，沒有daemon或背景作業
    • 建議沒事就不要去動它了
    • 絕對不要對他rm -rf .git，不然git的紀錄就會全部不見
  • master
    • master是目前所在分支的名稱，分支的概念會在以後的章節提到
    • 至於在terminal顯示分之名稱，則是oh-my-zsh-powerline-theme的功能

Stage

看到現在專案中各個檔案的狀態 

git status
不過我們已經透過alias把status設為st了，所以可以直接用 

git st
那麼接下來要介紹git很重要的四個stage

Stage

總共分成4個stage

  1. untracked

    (新的檔案，還沒加入git中)

  2. unmodified

    (完成add，但還沒commit)

  3. modified

    (add完後又做了修改)

  4. staged

    (commit完成，到了這個階段就已經完成紀錄)


接下來會介紹什麼是add和commit

Stage 3 - staged

所有的東西目前都已經加入git (因為根本就還沒有東西)

Stage 0 - untracked

  • 建立一個檔案叫hi.txt
  • 現在有一個檔案進入untracked

Stage 1 - unmodified

將新的檔案加入git 

git add "file name"

Stage 1 - untracked

如果一次要加入很多檔案怎麼辦？

Stage 1 - unmodified


git add .
不過一般建議還是一個一個檔案add比較好
避免add到錯誤的檔案

Stage 3 - staged

讓剛剛的檔案進入staged階段吧，之後這個檔案就會一直被記錄在git 

git commit
接著會進入編輯器，讓你輸入commit 預設的編輯器通常不是vi，可以透過這個指令設定 

git config --global core.editor=vim

Log

既然檔案已經進入紀錄
讓我們來查看之前的紀錄 

git log

Stage 2 - modified

如果修改了add或commit的檔案，就會變成這樣
這時候只要在用git add，加入紀錄就好了

Stage

  • 新增檔案 → 加入track (git add) → 提交(git commit) → 繼續加入其他檔案
  • 修改檔案 → 加入stage (git add) → 提交(git commit) → 繼續修改其他檔案
