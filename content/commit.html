How to write a good git commit

  • Priciple
    • 為每個不同的修改做一次commit
    • 不要一個commit中記錄太多的修改
    • 讓你的commit說一個故事
  • More Specific
    • 第一行應該在80字以下
    • 使用現在式
    • 不要去重複任何可以在git diff中被找到的東西
    • 更多的細節，應該空下第二行，從第三行開始寫
    • 在第三行以後，提到為什麼做了這次的修改
